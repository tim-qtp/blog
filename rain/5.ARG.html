# 一、为什么不直接使用网页版DeepSeek？

  • 我们的需求：绝对的隐私保护和个性化知识库构建
  • 场景：比如如果你希望大模型能根据你们企业的规章制度来回答问题，那么首先你肯定需要将这些规章制度或者模型作为附件上传，但你仍然可能面临一些别的问题：
    • 一个是数据隐私问题；联网使用大模型，所有数据均会上传到DeepSeek服务器，数据隐私性无法得到保证；
    • 第二个是上传文件的限制问题；一般来说网页版AI对于文件上传数量是有限制的，因为解析附件需要额外的算力；即使你是付费用户，有时你想要上传的文件不是一个两个，而是几百个文件，也就是一个专业领域的知识库，模型同样可能无力支持；
    • 第三个问题是上传文件附件方式使用起来繁琐：仅仅只是将文件作为附件加入对话上下文，每一次想要让模型根据这些附件回答问题的时候，都需要重新上传附件；想要新增、删除、修改已有的附件，同样也是很难实现的；

# 二、如何实现网页版DeepSeek不能实现的需求？

  • 隐私保护
    • 通过对话大模型（如DeepSeek）的本地部署解决隐私问题；
  • 个性化知识库构建 使用RAG技术（Retrieval-Augmented Generation，检索增强生成）构建个人知识库。为此我们需要：
    • 本地部署RAG技术所需要的开源框架RAGFlow；
    • 本地部署Embedding大模型（或者直接部署自带Embedding模型的RAGFlow版本）；

# 三、为什么要使用RAG技术？RAG和模型微调的区别？

  • 大模型不知道你的这些私有知识，当它回答自己不知道的问题时会出现“幻觉”问题；

  • 微调技术和RAG技术的区别：

    • 微调：在一个预训练模型（比如 GPT）基础上，使用特定领域的数据（比如医学、法律等）继续训练，使得模型更擅长这个领域的问题。（微调是考前复习，模型通过训练，消化吸收了这些知识然后给你回复）；
    • RAG：模型在生成回答前，会从外部知识库（数据库、文档、网页等）检索相关信息，结合检索到的内容生成更准确的回答。（RAG是开卷考试，模型看到你的问题，开始翻你的知识库，以实时生成更准确的答案）；

  • RAG（Retrieval-Augmented Generation）技术原理：

    # 1️⃣ 检索（Retrieval）→ 就是“去找资料”

    系统先不急着回答，它会去外部的“知识库”里翻资料，比如文档、数据库、网页等，看看有哪些内容可能和你的问题有关。 📚 类比：你问“如何养猫？”，它去翻了一本《猫咪饲养手册》。

    # 2️⃣ 增强（Augmentation）→ 就是“整理资料+问题一起考虑”

    它把找到的资料内容，加在你的问题后面，一起送给语言模型。 📎 类比：它把“你问的问题 + 猫咪手册的相关段落”打包给大脑看。

    # 3️⃣ 生成（Generation）→ 就是“根据问题+资料，写出回答”

    然后语言模型才开始写回答。因为它看过相关资料了，所以回答得更有根据、更准确。 📝 类比：它不是凭空编的，而是“看完书再写”的。

# 四、什么是Embedding？为什么除了DeepSeek、RAGFlow外我还需要“Embedding模型”？

  • 检索（Retrieval）：当用户提出问题时，系统会从外部的知识库中检索出与用户输入相关的内容。
    1. 准备外部知识库：外部知识库可能来自本地的文件、搜索引擎结果、API 等等。这样，RAG 不仅依赖于模型本身的内在的、通过训练得到的知识，还能实时调用外部的信息进行补充。我们要构建的就是一个基于本地文件的外部知识库；
    2. 通过 embedding 模型，对知识库文件进行解析：文本是由自然语言组成的，这种格式不利于机器直接计算相似度。embedding 模型要做的，就是将自然语言转化为高维向量，然后通过向量来捕捉到单词或句子背后的语义信息。比如，“蟹堡王”“比奇堡”“深度学习”三个词，embedding 会将蟹堡王和比奇堡映射到相近的向量空间中；而深度学习会被映射到离他们较远的向量空间中，这样通过embeding，机器就可以学习到自然语言之间的深度语义关系，并且依此计算出不同文本之间的相似关系了；
    3. 通过 embedding 模型，对用户的提问进行处理：用户的输入同样会经过嵌入（Embedding）处理，生成一个高维向量。
    4. 拿用户的提问去匹配本地知识库：使用这个用户输入生成的这个高维向量，去查询知识库中有无相关的文档片段。系统会利用某些相似度度量（如余弦相似度）去判断这个相似度。这一整个过程可以理解为，上传并解析了知识库之后，相当于给知识库中每一个小的文段都生成了一个指纹；然后用户输入问题之后，这个问题同样也会生成一个独一无二的指纹，接着RAGFlow系统就会拿着用户输入的这个指纹，在指纹库也就是知识库中匹配，找到相似的指纹，然后把将检索到的这些相关文段与用户的输入结合，扩展模型的上下文，再喂给对话模型DeepSeek。
  • 模型的分类：Chat模型、Embedding模型；
  • 简而言之：Embedding模型是用来对你上传的附件进行解析的；
